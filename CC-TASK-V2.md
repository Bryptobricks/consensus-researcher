## 1. File-Based Cache Layer

Add a cache system so identical queries don't burn Brave API credits.

### Implementation:
- Cache directory: `data/cache/` (create if not exists, add `data/cache/.gitkeep`)
- Cache key: MD5 hash of `query + category + depth` → `{hash}.json`
- Default TTL: 30 minutes
- Quick mode TTL: 2 hours
- Each cache entry stores: `{ query, category, depth, timestamp, result }`

### Cache functions:
```js
function cacheKey(query, category, depth) // MD5 hash → 12-char hex
function cacheGet(query, category, depth, ttlMs) // returns result or null
function cacheSet(query, category, depth, result) // writes to file
function cachePrune(ttlMs) // delete expired entries
function cacheClear() // delete all
```

### CLI integration:
- Before making any API calls in `runResearch()`, check cache. If hit, log `[research] (cached — skipping API calls)` and return cached result.
- Add `--no-cache` flag to force fresh fetch
- Add cache subcommand: `node research.js cache clear` and `node research.js cache prune`
- Use Node built-in `crypto.createHash('md5')` — zero deps

## 2. Auto-Save to Research Directory

Add `--save` flag that auto-generates a filename and saves the full result + a human-readable markdown summary.

### Implementation:
- Save directory: configurable, default `./memory/research/` (relative to cwd)
- Filename: `{slug}-{YYYY-MM-DD}.md` where slug is query sanitized to lowercase alphanumeric + hyphens, max 40 chars
- The saved file should be a readable markdown report, NOT raw JSON:

- Also save the raw JSON alongside as `{slug}-{YYYY-MM-DD}.json` for programmatic access

## 3. Cost Tracking

Display API call count and estimated cost after every run.

### Implementation:
- Track every Brave Search API call and Reddit JSON fetch in a counter object:
```js
const apiCalls = { brave: 0, reddit: 0, total: 0 };
```
- Increment in `braveSearch()` and `fetchRedditThread()`
- At end of `runResearch()`, log to stderr:
```
📊 API calls: 5 Brave + 3 Reddit = 8 total | Est. cost: ~$0.025
```
- Brave cost: $5 per 1,000 queries = $0.005/query
- Reddit JSON: free (no API key)
- Also include in the output JSON as a new `apiCost` field:
```json
"apiCost": {
  "braveCalls": 5,
  "redditCalls": 3,
  "totalCalls": 8,
  "estimatedUSD": 0.025
}
```

## 4. Product Watchlist

Persistent list of products/brands to monitor. Designed for heartbeat integration — the agent can run `research.js watchlist check` periodically to see if anything changed.

### Implementation:
- Watchlist file: `data/watchlist.json`
- Schema:
```json
{
  "items": [
    {
      "query": "Nutricost glycine powder",
      "category": "supplement",
      "note": "Current daily supplement",
      "addedAt": "2026-03-06T20:00:00Z",
      "lastChecked": "2026-03-06T20:00:00Z",
      "lastScore": "HIGH"
    }
  ]
}
```

### CLI subcommands:
```bash
node research.js watchlist                              # Show all items
node research.js watchlist add "query" [--note "..."]   # Add item
node research.js watchlist remove "query"               # Remove item
node research.js watchlist check                        # Quick-research all items, report changes
```

### `watchlist check` behavior:
- Run a `quick` depth research on each item
- Compare data sufficiency and source count to last check
- Report to stdout:
```
📋 Watchlist Check (3 items)

✅ Nutricost glycine powder — no change (HIGH, 3 Reddit threads)
⚠️ Sports Research D3+K2 — new Reddit activity (2 new threads since last check)
✅ Thorne creatine — no change (HIGH, stable)
```
- Update `lastChecked` timestamp after each check

## 5. Comment Score Filter

Add `--min-score N` flag to filter out low-quality Reddit comments.

### Implementation:
- After fetching and parsing Reddit comments, filter out any with `score < N`
- Default: no filter (show all comments)
- Example: `--min-score 5` removes all comments with fewer than 5 upvotes
- Apply BEFORE sorting and BEFORE the result is cached
- Log how many were filtered: `[research] Filtered 47 comments below score 5 (78 remaining)`
